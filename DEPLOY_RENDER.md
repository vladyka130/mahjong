# Інструкція для деплою на Render.com

## Покрокова інструкція:

### 1. Підготуйте проект
- Переконайтеся, що всі файли закомічені та запушені на GitHub
- Репозиторій: https://github.com/vladyka130/mahjong.git

### 2. Створіть акаунт на Render.com
- Перейдіть на https://render.com
- Зареєструйтеся або увійдіть
- Підключіть свій GitHub акаунт

### 3. Створіть новий Web Service
1. Натисніть "New +" → "Web Service"
2. Підключіть репозиторій `vladyka130/mahjong`
3. Налаштування:
   - **Name**: `mahjong-solitaire` (або будь-яка назва)
   - **Environment**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python main_flet.py`
   - **Plan**: Free (або платний, якщо потрібно)

### 4. Налаштуйте Environment Variables (опціонально)
- `RENDER=true` - для автоматичного визначення середовища

### 5. Деплой
- Натисніть "Create Web Service"
- Render автоматично почне деплой
- Чекайте завершення (зазвичай 5-10 хвилин)

### 6. Перевірка
- Після успішного деплою отримаєте URL типу: `https://mahjong-solitaire.onrender.com`
- Відкрийте URL у браузері

## Важливі примітки:

1. **Безкоштовний план Render.com**:
   - Сервіс "засинає" після 15 хвилин неактивності
   - Перший запуск може займати 30-60 секунд
   - Обмеження ресурсів

2. **База даних**:
   - Файл `mahjong_db.enc` буде створюватися автоматично
   - Дані зберігаються локально на сервері
   - При перезапуску сервісу дані можуть втратитися (на безкоштовному плані)

3. **Порт**:
   - Додаток налаштований на порт 8000
   - Render автоматично призначає порт через змінну `PORT`

## Альтернатива: Використання змінної PORT

Якщо Render вимагає використання змінної PORT, можна оновити код:

```python
import os
port = int(os.getenv("PORT", 8000))
ft.app(target=main, view=ft.AppView.WEB_BROWSER, port=port, host="0.0.0.0")
```
